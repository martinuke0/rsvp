<task type="auto">
  <name>Initialize shadcn/ui with Stone theme</name>
  <files>
    components.json
    lib/utils.ts
    app/globals.css
  </files>
  <action>
Run `npx shadcn@latest init` with the following selections when prompted:

1. "Would you like to use TypeScript?" → Yes
2. "Which style would you like to use?" → Default
3. "Which color would you like to use as base color?" → **Stone** (critical for granite aesthetic)
4. "Where is your global CSS file?" → app/globals.css
5. "Would you like to use CSS variables for colors?" → Yes
6. "Where is your tailwind.config.js located?" → tailwind.config.ts
7. "Configure the import alias for components?" → @/components
8. "Configure the import alias for utils?" → @/lib/utils

This creates:
- `components.json` - shadcn configuration file
- `lib/utils.ts` - Utility functions (cn for className merging)
- Updates `app/globals.css` with Stone theme CSS variables
- Updates `tailwind.config.ts` with shadcn plugin

Verify that `app/globals.css` contains Stone color palette CSS variables (neutral, earthy tones with low saturation).
  </action>
  <verify>
Check that `components.json` exists with `"baseColor": "stone"` and `"cssVariables": true`.

Check that `app/globals.css` contains CSS variables like `--background`, `--foreground`, `--primary` with Stone palette values.

Check that `lib/utils.ts` exists and exports `cn` function.
  </verify>
  <done>
shadcn/ui is initialized with Stone theme, CSS variables are configured in globals.css, and utility functions are available for component styling.
